# 使用说明书

## 1. 启动

在项目根目录双击 `启动助手.bat`，或在 PowerShell 中运行：

```powershell
cd <项目目录>
.\.venv\Scripts\zju-seat-assistant.exe --host 127.0.0.1 --port 8765 --data-dir data
```

打开页面：

```text
http://127.0.0.1:8765
```

## 2. 连接账号

点击“连接浙大账号”。连接账号时会打开可见浏览器，便于输入账号、密码和完成二次认证。

程序每 20 秒检查一次登录状态。完成登录后，可以点击“我已完成登录”立即验证。

## 3. 创建任务

点击“新建任务”，依次选择：

- 馆舍、楼层、分区；
- 预约日期和使用时间；
- 自动检查开始、停止时间；
- 刷新间隔和最大连续错误次数；
- 座位规则；
- 是否发现空位后直接预约；
- 是否在成功、超时、异常时弹出屏幕提示。

馆舍、楼层、分区支持单选或多选。多选分区保存时会自动拆分成多个任务。

## 4. 自动提交

默认不会直接预约。需要同时满足两个条件才会点击“立即预约”：

1. 在系统设置中开启“允许任务自动提交预约”，并输入确认短语 `确认自动预约`。
2. 在任务中勾选“发现空位后直接预约”。

如果任一条件不满足，程序只检测并在页面显示候选座位，不会提交预约。

## 5. 屏幕通知

本版本不再使用 SMTP 邮件通知，也不需要邮箱授权码。

任务中有三个屏幕通知开关：

- 预约成功时弹出屏幕提示；
- 任务超时时弹出屏幕提示；
- 任务异常时弹出屏幕提示。

Windows 上使用系统自带弹窗，默认 10 秒自动关闭。网页中的“最近检测”“检测结果”“候选座位”“下次检测”仍会实时显示。

## 6. 后台浏览器

预约检测和选座默认使用后台浏览器，避免影响正常使用电脑。

调试时如需显示预约浏览器，可在启动前设置：

```powershell
$env:ZJU_SEAT_RESERVATION_VISIBLE = "true"
.\.venv\Scripts\zju-seat-assistant.exe --host 127.0.0.1 --port 8765 --data-dir data
```

## 7. 本地数据

- `data/assistant.db`：任务、设置、状态和预约结果。
- `data/browser-profile/`：独立浏览器登录状态，不要上传或分享。
- `data/diagnostics/`：异常截图，分享前检查是否包含个人信息。

## 8. 测试

```powershell
.\.venv\Scripts\python.exe -m pytest -q
```
