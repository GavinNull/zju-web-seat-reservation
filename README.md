# ZJU Web Seat Reservation Assistant

浙江大学图书馆网页版座位助手。  
这是一个运行在 Windows 本地的 Web 控制台，通过 Playwright Chromium 操作真实的图书馆预约网页，帮助用户配置座位检测、候选座位筛选、后台刷新和可选的自动预约。

> 本项目只做本机自动化辅助，不保存统一身份认证密码，不绕过验证码、二次认证、限流或学校预约规则。请遵守浙江大学图书馆相关规定。

## 项目状态

- 真实 UI 流程已测试：登录、检测、选座、预约成功。
- 预约检测默认在后台窗口运行，不跳到前台打断电脑使用。
- 无空位时会停留在当前座位选择页刷新，不反复退出重进。
- 通知方式为本机屏幕提示。
- 任务卡片会显示当前步骤、检测轮次、最近过程和最终结果。

## 快速开始

推荐直接双击：

```text
启动助手.bat
```

首次运行时，启动助手会自动完成这些准备工作：

- 切换到项目目录。
- 创建 `.venv` Python 虚拟环境。
- 安装项目依赖。
- 安装 Playwright Chromium。
- 启动本地服务并打开 `http://127.0.0.1:8765`。

如果自动安装失败，再按下面的 PowerShell 命令手动执行。

进入项目目录：

```powershell
cd <项目目录>
```

检查 Python 版本，建议 Python 3.11 或更高：

```powershell
python --version
```

创建虚拟环境：

```powershell
python -m venv .venv
```

安装项目依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -e ".[test]"
```

安装 Playwright Chromium：

```powershell
.\.venv\Scripts\playwright.exe install chromium
```

启动服务：

```powershell
.\.venv\Scripts\zju-seat-assistant.exe --host 127.0.0.1 --port 8765 --data-dir data
```

打开控制台：

```text
http://127.0.0.1:8765
```

## 环境依赖

推荐环境：

- Windows 10 或 Windows 11。
- Python 3.11 或更高版本。
- PowerShell 或 Windows Terminal。
- 可访问浙江大学图书馆预约网站的校园网、VPN 或校内网络环境。
- 可正常完成浙大统一身份认证，包括密码、短信、App 或其他二次认证方式。

Python 运行依赖由 `pyproject.toml` 管理，安装项目时会自动安装：

- `FastAPI`：本地 Web 控制台服务。
- `uvicorn[standard]`：本地 HTTP 服务运行器。
- `Jinja2`：页面模板渲染。
- `APScheduler`：预约任务调度。
- `playwright`：控制 Chromium 操作预约网页。
- `Pillow`：截图和图像相关能力。
- `python-multipart`：请求解析支持。

测试依赖：

- `pytest`
- `httpx`

不需要：

- 不需要手动安装 Chrome 或 Edge。
- 不需要 Node.js、npm、Docker 或远程服务器。
- 不需要数据库服务；本地任务数据使用 SQLite 文件保存。
- 不需要 SMTP、邮箱账号或邮箱授权码。

## 功能特性

- 本地 Web 控制台，默认只监听 `127.0.0.1`。
- 使用独立 Playwright Chromium，不读取用户日常浏览器 Cookie。
- 支持馆舍、楼层、分区联动选择。
- 支持分区单选或多选，多选会拆分成多个任务；启动后每个区域使用独立的不可见预约窗口。
- 支持座位范围、指定座位、排除座位和优先级规则。
- 支持观察模式，只检测候选座位，不提交预约。
- 支持自动提交模式，在用户明确开启后点击“立即预约”。
- 支持无空位时停留在当前座位页刷新。
- 支持每轮检测完成后立即继续下一轮，实际速度由网页刷新和扫描耗时决定。
- 支持任务卡片实时显示当前步骤、检测轮次和最近过程。
- 支持预约过程后台窗口运行，减少对日常使用的影响。
- 支持预约成功、超时、异常的本机屏幕通知。
- 支持异常截图和诊断文本，方便排查网页变化。

## 使用流程

1. 启动本地服务并打开 `http://127.0.0.1:8765`。
2. 点击“连接浙大账号”。
3. 在弹出的浏览器中完成浙大统一身份认证。
4. 回到助手页面，确认账号状态为“已连接”。
5. 新建或编辑预约任务。
6. 选择馆舍、楼层、分区、日期、使用时间和座位规则。
7. 保持“观察模式”可只检测候选座位。
8. 如需自动预约，开启系统总开关，并在任务中关闭“观察模式”。
9. 点击“启动”，或点击“检查一次”立即执行检测。

## 正确登录流程

账号登录状态保存在本项目的本地浏览器目录中。第一次使用、换目录运行、清理 `data/` 后，或登录状态过期后，都需要重新连接浙大账号。

推荐按以下流程操作：

1. 从当前项目目录启动服务，确保使用同一个 `--data-dir data`。
2. 打开助手页面 `http://127.0.0.1:8765`。
3. 点击页面中的“连接浙大账号”。
4. 在弹出的浏览器窗口中完成浙大统一身份认证，包括密码、短信、App 或其他二次认证。
5. 登录完成后，等待浏览器跳转回浙江大学图书馆预约平台或预约系统首页。
6. 回到助手页面；如果页面没有自动更新状态，点击“我已完成登录”。
7. 确认页面显示账号已连接，再启动任务或点击“检查一次”。

注意事项：

- 登录必须在助手弹出的浏览器窗口中完成，普通 Chrome、Edge 或其他日常浏览器中的登录状态不会自动同步到本项目。
- 不要在统一身份认证还没有跳转完成前关闭登录浏览器。
- 如果任务显示“等待登录”或“需要重新连接浙大账号”，说明当前项目的 `data/browser-profile/` 中没有可用登录状态，按上面的流程重新连接即可。
- 如果反复提示未登录，先确认没有从另一个项目目录启动服务，也没有换用其他 `--data-dir`；必要时关闭旧服务后，从当前项目目录重新启动。
- `data/browser-profile/` 等同于本项目的本地登录凭据，只保存在自己的电脑上，不要上传 GitHub，也不要复制给他人。

## 自动预约开关

自动预约需要同时满足两个条件：

- 系统设置中勾选“允许任务自动提交预约”，输入确认短语 `确认自动预约` 并保存。
- 具体任务中关闭“观察模式”。

只要任意一个条件不满足，程序只会检测候选座位，不会点击“立即预约”。

## 后台运行策略

登录账号时会打开可见浏览器，因为需要用户手动输入账号、密码或完成二次认证。

预约检测和选座默认使用后台窗口运行。它不是纯 headless 模式，而是把真实 Chromium 窗口放到屏幕外执行，以尽量复用真实登录态，同时避免预约页面跳到最前面。

如需调试并显示预约浏览器，可以在启动服务前设置：

```powershell
$env:ZJU_SEAT_RESERVATION_VISIBLE = "true"
.\.venv\Scripts\zju-seat-assistant.exe --host 127.0.0.1 --port 8765 --data-dir data
```

## 无空位刷新策略

进入指定馆舍、楼层和分区的座位选择页后：

- 如果找到符合规则的空位，观察模式下只显示候选座位。
- 如果允许提交，程序会继续选座并点击预约。
- 如果暂时没有空位，程序不会退出座位选择页。
- 程序会在当前座位页刷新；每轮检测完成后立即进入下一轮，不再额外等待固定刷新间隔。
- 多个区域同时运行时，每个区域保留自己的座位选择页并独立刷新。
- 任一区域预约成功后，程序会停止其他区域任务并关闭其他预约窗口，避免重复预约。
- 到达任务停止时间仍未发现空位时，任务结束为无可用座位或超时状态。

## 本地数据与隐私

项目按本地优先方式设计：

- 服务默认只监听 `127.0.0.1:8765`。
- 不提供公网服务。
- 不主动上传任务、账号、Cookie、截图或预约记录。
- 不保存浙大统一身份认证密码；密码输入发生在学校登录页面中。
- 不使用邮件通知，因此不保存邮箱账号、SMTP 密码或邮箱授权码。

以下目录或文件只应保留在本机，不要上传 GitHub：

- `data/assistant.db`：任务、设置、状态和预约结果。
- `data/browser-profile/`：独立浏览器登录状态，等同于本项目的本地登录凭据。
- `data/diagnostics/`：异常截图和诊断文本，可能包含个人页面信息。
- `.venv/`：本地 Python 虚拟环境。
- `.pytest_cache/`、`__pycache__/`：测试和 Python 缓存。

本仓库的 `.gitignore` 已默认排除这些本地数据。
